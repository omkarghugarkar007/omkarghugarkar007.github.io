    Once you are done with the intro, then the real practise begins. Following are the important topics.

    1. Array
    It is one of the easiest topic but still many coding test has a question related to array. This topic would not require any specifc YouTube playlist as such. Directly practising Needcode questions would do.

    2. Two Pointer, Sliding Window and Intervals
    I would recommend to go through the Following YouTube playlist before trying the NeetCode questions. The topics are quite similar.
    Aditya Verma Sliding Window
    You can first try the questions mentioned in the video than the Needcode questions. Most of the questions would be common.

    3. Stack, Binary Search and Heap
    Same approach as [2] can be followed.
    Aditya Verma Stack
    Aditya Verma Binary Search
    Aditya Verma Heap

    4. Linked List and Greedy
    Whatever you learnt in the Udemy course should be sufficient for you to do the NeetCode questions

    5. Recursion
    Probably the most important topic. Following are the two playlists I recommend. You can go with anyone of them and then later try the backtracking section from NeetCode
    Aditya Verma Recursion
    Striver Recursion

    6. Dynamic Programming
    You might be spending your most amount of time of this topic. Need to practice lots of questions. Following are the two playlists I recommend. Again, you can go with anyone of them but I would recommend to do both.
    Aditya Verma DP
    Striver DP

    7. Trees
    BFS and DFS learnt in the Udemy course and DP on trees would be sufficient for the NeetCode questions

    8. Graphs
    One of the advance topics and usually done at the last. Following playlist would be more than sufficient.
    Striver Graphs

    Disclaimer:
    These topics would actually not be sufficient to do all the questions in the coding test. Many CP related problems are also asked, though I found them more in Placement tests as compared to Intern tests. Such topics are infinite in nature and can only be applied to few questions.
    If you are able to complete the above 8 topics, more than 75% questions in the coding test would be easy for you then :)

  • Software Development Roles

    Along with DSA, various other topics are very much essential. Mostly asked in interviews, but some companies can ask MCQ based questions or SQL queries. I would recommend to do this preparation parallel with DSA.

    1. SQL
    Many companies ask direct SQL queries in the placements test. I found that questions asked in the test are generally not very straight forward (though it is considered to be one one of the easiest topic) and a good practise of SQL queries is necessary. Many videos and playlists are available on YouTube. You can go with this SQL. You can practise questions on HackerRank or Leetcode but make sure to do "Joins" well.

    2. CS concepts
    MCQ based questions possible in the tests. Direct concepts can be asked in the interviews. You can refer to the following series.
    a. DBMS Babbar DBMS
    b. Computer Networks GateSmashers CN
    c. Operating Systems Babbar OS

    Major focus should be on DSA and SQL. Parellely, you can do these topics.

    Does language matter for DSA?
    5-10% of companies would expect you to code in a specifc langauge, but except that you should go with the language of your convinence.

    Interview Preparation
    Before the interviews, you should try doing some mock interview rounds and have a habit of writing code using pen and paper. You should try to keep the interviewer involved in whatever you are thinking or writing on paper.

  • Data Science and Machine Learning Roles

    I am assuming you are not new to this field and have already done some prerequite courses like Andrew NG from coursera or some similar courses. You should also have some good projects on your resume. The interviews mostly revolves arounds the projects you have done and some basic Machine learning concepts.

    ML concepts
    1. Linear and Logistic Regression
    2. SVM
    3. Bagging/Boosting
    4. KNN
    5. Clustering
    6. Navie Bayes
    7. Performance Metrics
    8. Optimizers
    9. Loss functions
    10. Basic understanding of Neural Networks and CNN's

    You should be thorough with these topics. You can find various videos on these topics on YouTube. I would recommend Krish Naik for these topics.

    Following is a cheatsheet which would be very helpful - Cheatsheet
    Revising these concepts before the tests starts would be very useful as many companies ask MCQ questions.

    Projects
    One can expects the following questions.
    1. What are the input and output of the model?
    2. Which model was used? Why?
    3. What was unique in this model?
    4. What were your performance metrics and what you can do to improve this?

    Don't forget to read in-depth if you are using a very standerd Model like Yolo, BERT, etc. You can easily find blogs on Medium.
    I would recommend to make a doc for each project in your resume which has details about the project and the answers to above questions.
